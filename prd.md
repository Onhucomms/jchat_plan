# Product Requirements Document (PRD)
## 데이팅 앱 — Onhucomms 기반 확장

> 기존 Onhucomms 채팅 서비스를 데이팅 앱으로 재정의·확장한다.  
> 정책 → 비즈니스/기능 → UI 순으로 정의하며, AI Agent 활용 및 기존 기술 스택을 유지한다.

## 1. 정책 (Policy)

### 1.1 서비스 정책

#### 서비스 목적
- **문제 인식**: 기존 데이팅 앱은 남성 사용자가 여성보다 우세해 성비가 맞지 않으며, 이로 인해 서비스 활용도와 만족도가 떨어진다.
- **본 서비스의 목적**:
  1. **성비 균형**: 남/녀 비율을 일정 수준으로 맞춰 데이팅 앱의 활용도를 높이고, 상대적으로 균형 잡힌 환경을 제공한다.
  2. **수익 창출**: 성비가 맞지 않아 일반 가입(무료·대기)으로는 가입이 어려운 경우, **포인트를 사용해 가입·승인**할 수 있게 하여 수요를 수익으로 전환한다.
- **정리**: 성비 맞춤으로 서비스 가치를 높이고, 성비 제한으로 인한 “가입 대기” 상황을 포인트 결제로 연결해 수익을 창출하는 것이 이 앱의 핵심 목적이다.

#### 타깃·포지셔닝
- 성인(20세 이상) 데이팅·채팅 이용자.
- 성비가 맞는 환경에서의 대화·만남을 원하는 사용자.
- 빠른 가입(바로승인)이 필요한 사용자 → 포인트 결제 유도 대상.

#### 이용약관 방향
- (추후 대화를 통해 구체화)

### 1.2 회원/가입 정책

#### 가입 자격·연령
- **20세 이상 성인만** 회원가입 가능하다.

#### 회원 단계
- 회원은 다음 두 단계로 구분한다.
  - **대기회원**: 가입 신청은 완료했으나, 성비상 서비스 이용(정회원 전환) 승인이 나지 않은 상태. 채팅 등 본 서비스 이용 불가.
  - **회원**: 승인 완료되어 서비스를 이용할 수 있는 상태.

#### 성비에 따른 가입·승인 방식
- **바로 가입(즉시 회원)**: 현재 남·녀 수가 **같을 때** 해당 성별로 가입하면 별도 대기 없이 바로 **회원**으로 가입된다.
- **대기회원으로 가입**: 성비가 맞지 않아 당장 정원이 없을 때(예: 남자가 여자보다 많은 상태에서 남자가 가입)에는 **대기회원**으로만 가입된다.
- **자동 승인(정회원 전환)**: 상대 성별(여성 또는 남성)이 새로 가입·승인되면, 대기회원 중에서 **추가로 자동 승인**하여 **회원**으로 전환한다. 이렇게 남·녀가 번갈아 가며 승인되는 형태로 성비를 맞춰 나간다.
  - 예: 남자가 먼저 가입해 대기회원이 되면, 그다음 여자가 가입·승인될 때 그 남자 1명이 자동으로 회원 전환; 이후 남자가 한 명 더 가입하면 대기회원, 그다음 여자 1명 가입 시 그 남자가 자동 회원 전환.

#### 본인인증
- **방식**: KCP 등 **외부 본인인증 서비스**를 연동하여 본인인증을 수행한다.
- **수신·활용 정보**: 본인인증 완료 후 해당 서비스에서 전달받은 다음 정보를 이용한다.
  - **성별**: 성비 정책 적용(대기회원/회원 구분, 자동 승인 순서) 및 서비스 내 성별 구분에 사용한다.
  - **나이**: 20세 이상 여부 판단 및 가입 자격 검증에 사용한다.
  - **핸드폰 번호**: 동일인 **연속가입(중복가입) 방지**에 사용한다. 동일 휴대폰 번호로의 재가입은 제한한다.
- 위 정보(성별, 나이, 휴대폰번호)는 암호화 등 개인정보 보호 방안에 따라 저장·관리하며, 구체적인 저장 방식은 기술 요구사항에서 정리한다.

#### 가입 플로우
가입은 아래 순서대로 진행하며, 모든 단계를 완료하면 가입이 완료된다.

1. **디바이스·계정 인증**
   - **안드로이드**: Google 인증을 이용한다.
   - **아이폰(iOS)**: Apple 인증을 이용한다.
2. **본인인증**
   - KCP 등 외부 본인인증 서비스를 통해 **성별**, **연령**, **핸드폰 번호**를 접수한다. (가입 자격·성비·중복가입 방지에 사용)
3. **프로필 입력**
   - **닉네임**: 필수 입력.
   - **나를 표현하는 한마디**: 필수 입력.
   - **프로필 사진**: 선택(옵션).
   - **원하는 만남 형태**: 추후 항목·선택지 정의 후 기입. (현재는 필드만 준비)

위 1 → 2 → 3 순서대로 입력이 완료되면 가입이 완료되며, 이때 성비 정책에 따라 **회원** 또는 **대기회원**으로 최종 확정된다.

### 1.3 이용 정책
- **채팅 가능 리스트**: 채팅 가능한 리스트에는 **반대 성별의 회원**만 노출된다. (남성 사용자 → 여성 회원 리스트, 여성 사용자 → 남성 회원 리스트)
- 신고·제재, 프라이버시·위치 정책 등은 추후 구체화한다.

### 1.4 수익/포인트 정책
- **채팅 요청 비용**: 회원(대기회원 제외)이 상대방에게 채팅(대화) 요청을 보낼 수 있으며, **50포인트는 상대방이 채팅을 시작했을 때만** 차감된다. 상대방이 채팅을 시작하지 않으면 포인트는 차감되지 않는다. 보유 포인트가 50 미만이면 대화 요청을 할 수 없다.
- **채팅 시작 정의**: 해당 채팅방에서 **입력된 글자가 20글자를 넘어야** 채팅을 시작한 것으로 본다. 20글자 이하이면 채팅 시작으로 보지 않으며, 이 경우 50P가 차감되지 않는다.
- **포인트 충전 경로**: 무료충전소(제휴 업체), 포인트 충전(결제)을 통해 충전 가능. (상세는 2.3 참조)
- **상대방 포인트 선물**: 회원이 채팅 상대방에게 보유 포인트를 선물할 수 있다. (선물 단위·한도 등은 추후 구체화)
- 포인트 적립·환불 등 추가 정책은 추후 구체화한다.

## 2. 비즈니스/기능 요구사항

### 2.1 회원

> 가입 자격, 회원 단계(대기회원/회원), 성비·승인 방식, 본인인증, 가입 플로우는 **1.2 회원/가입 정책**을 따른다.

#### 회원 상태
회원(및 대기회원)은 다음 상태 값을 가진다. 서비스 이용 가능 여부와 채팅 리스트 노출은 이 상태에 따른다.

| 상태 | 설명 |
|------|------|
| **online** | 앱을 최근에 사용 중인 회원. 채팅 가능 리스트에 노출된다. |
| **offline** | 일정 기간 앱 미사용으로 전환된 회원. 채팅 가능 리스트에서 **제외**된다. |
| **사용불가** | 신고 누적 등으로 서비스 이용이 제한된 상태. |
| **강제탈퇴** | 사용불가 반복 등으로 계정이 탈퇴 처리된 상태. |
| **본인탈퇴** | 회원이 직접 탈퇴한 상태. |

- **online → offline**: online 회원이 **앱을 3일간 구동하지 않으면** 자동으로 offline 처리된다.
- **offline → online**: offline 회원이 **앱을 다시 구동하면** 자동으로 online 처리된다.

#### 신고·제재
- 회원이 채팅 상대에게 **신고를 누적 3회** 받으면 **사용불가** 처리된다.
- **사용불가 상태를 3회** 받으면(즉, 사용불가로 전환된 뒤 또 신고 등으로 사용불가가 누적되면) **강제탈퇴** 처리된다.  
  *(구체적인 “사용불가 3회” 정의는 운영 정책에서 보완 가능)*

#### 가입 완료 후 화면(승인 대기 시)
- 가입 완료 시점에 **대기회원**인 경우:
  - **승인 대기 인원 수**를 표시한다.
  - 포인트를 이용한 **"바로승인"** 버튼을 노출하여 포인트 충전을 유도한다.
  - **바로승인** 클릭 시 **200포인트를 차감**하고, 해당 회원을 즉시 **회원**으로 승인 처리한다.

#### 포인트(회원 관련)
- **대기회원·회원 모두** 포인트를 누적·보유할 수 있다.
- **승인 완료**(대기회원 → 회원 전환) 시 서비스 이용을 위해 **500포인트를 부여**한다.

#### 프로필 관리
- **프로필 설정** 화면에서 회원이 다음 정보를 **변경**할 수 있다.
  - 닉네임, 나를 표현하는 한마디, 프로필 사진, 원하는 만남 형태  
  (가입 시 입력한 항목과 동일, 변경 가능 범위는 정책·UI에서 정의)

#### 앱 실행 시 위치
- 앱 **로딩 시** 현재 위치 정보를 추적하여 사용자 **현재 위치**를 설정한다.
- 사용자 현재 위치는 **5분에 한 번** 조회·갱신하여 위치 기반 정렬(거리순) 등에 사용한다.

#### 참고(기존 Product.md와의 정리)
- 로그인·가입 경로(Google/Apple), 본인인증(KCP), 가입 단계(대기회원/회원) 및 성비 규칙은 PRD 1.2 기준으로 하며, 기존 문서의 메일 인증·최초 50/50 규칙은 현재 PRD에서는 적용하지 않는다.

### 2.2 매칭·채팅

#### 채팅 가능 리스트
- 리스트에는 **반대 성별의 회원** 중 **online 상태인 회원만** 노출된다. (offline 회원은 2.1 회원 상태에 따라 제외)
- **리스트 한 건에 표시되는 정보**:
  - 썸네일(프로필 사진, 없을 경우 기본 이미지)
  - 닉네임
  - 나를 표현하는 한마디
  - 원하는 만남 형태
  - 현재까지 진행한 채팅 횟수
  - 좋아요 횟수

#### 리스트 정렬
- **기본 정렬**: 가까운 거리순.
- **상단 필터**로 사용자가 정렬을 변경할 수 있다.
  - 거리순(가까운 순)
  - 채팅 횟수순
  - 좋아요 순

#### 만남 형태 필터
- 리스트 **상단**에 **원하는 만남 형태**를 버튼 형태로 노출한다.
- 사용자가 특정 만남 형태 버튼을 선택하면, **해당 만남 형태를 지정한 회원만** 리스트에 노출된다.
- **전체** 버튼을 두어, 선택 시 만남 형태 관계없이 전체 회원이 노출되도록 한다.

#### 매칭 방식
- **지정 매칭**: 사용자가 리스트에서 상대를 골라 직접 채팅을 시작하는 방식. (본인이 지정하여 매칭)
- **랜덤 만남**: 하단 메뉴에서 **랜덤으로 만날 수 있는 기능**을 제공한다. 선택 시 랜덤으로 상대와 매칭되어 대화할 수 있게 한다.

#### 채팅(대화) 요청 시 포인트
- **회원**(대기회원 아님)이 상대방에게 채팅(대화) 요청을 보낼 수 있다. **50포인트는 상대방이 채팅을 시작했을 때만** 차감된다. 상대방이 채팅을 시작하지 않으면 포인트는 차감되지 않는다.
- **채팅 시작 요건**: 해당 채팅방에서 **입력된 글자가 20글자를 넘어야** 채팅을 시작한 것으로 인정한다. 20글자 이하이면 채팅 시작이 아니며, 이 경우 요청자에게 50P가 차감되지 않는다.
- 회원이 보유한 포인트가 **50 미만**이면 상대방에게 **대화 요청을 할 수 없다**. (요청 버튼 비활성화 또는 안내 처리)

#### 채팅 데이터 저장
- **채팅 내용**은 **사용자 단말(폰) 내부 데이터베이스**에 저장되어야 한다. 즉, 채팅 메시지는 해당 사용자의 기기 로컬 DB에 보관하며, 앱에서 로컬 저장소를 사용해 조회·표시한다. (서버에 채팅 내용을 상시 보관할지 여부는 별도 정책·기술 요구사항에서 정의)

### 2.3 포인트·결제

#### 포인트 충전
- **무료충전소**: 외부 **제휴 업체**를 이용하여 무료로 포인트를 충전할 수 있는 경로를 제공한다.
- **포인트 충전(결제)**: 유료 결제로 포인트를 충전한다.
  - **현재**: 앱 마켓(Google Play / App Store) **PG 시스템**을 이용한다.
  - **추후**: 외부 PG를 이용한 결제를 연동한다.

#### 포인트 사용(요약)
- 바로승인: 200P 차감 (1.2·2.1 참조)
- 채팅(대화) 요청: 상대방이 채팅 시작 시 50P 차감 (채팅 시작 = 해당 방 입력 글자 20글자 초과, 1.4·2.2 참조)
- 상대방 포인트 선물: 보유 포인트를 채팅 상대방에게 선물 (1.4 참조, 단위·한도 추후 정의)
- 승인 완료 시 500P 지급 (2.1 참조)

## 3. UI/UX 요구사항

### 3.1 채팅 리스트 화면
- **리스트 항목**: 썸네일, 닉네임, 나를 표현하는 한마디, 원하는 만남 형태, 채팅 횟수, 좋아요 횟수를 표시한다.
- **상단 — 정렬 필터**: 거리순 / 채팅 횟수순 / 좋아요 순 선택 가능.
- **상단 — 만남 형태 필터**: 원하는 만남 형태를 버튼으로 배치하고, 선택 시 해당 만남 형태를 지정한 회원만 노출. **전체** 버튼 포함.

### 3.2 하단 메뉴
- **랜덤 만남** 기능을 하단 메뉴에 배치하여, 탭/버튼 선택 시 랜덤 매칭으로 대화할 수 있게 한다.

### 3.3 채팅창 화면 (진행 중인 채팅)

#### 상단
- **채팅방 제목**: 상단에 **"상대방 닉네임"님과 대화** 형식으로 표시한다. (예: "홍길동님과 대화")
- **뒤로가기(back 화살표)**: 상단 back 화살표 터치 시 **확인(confirm) 창**을 띄운다.
  - 문구: **"채팅방에서 나가시겠습니까?"**
  - **확인** 선택 시에만 채팅창에서 나간다. 취소 시 채팅창 유지.
- **신고**: 상단에 신고 기능을 두어, 해당 채팅 상대를 신고할 수 있다.
- **상대방에게 선물 보내기**: 상단에 상대방에게 포인트 선물을 보내는 기능을 둔다.

#### 하단
- **메시지 입력창**: 메시지를 입력하는 필드.
- **메시지 전송 버튼**: 입력한 메시지를 전송하는 버튼.
- **이미지 첨부**: 이미지를 첨부하여 전송할 수 있는 기능.
- **이모지 첨부**: 이모지를 첨부하여 전송할 수 있는 기능.
